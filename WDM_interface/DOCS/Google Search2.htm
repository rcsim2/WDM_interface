Martin Maurer wrote:
> first: is there a newsgroup for WDM related questions ?

comp.os.ms-windows.programmer.nt.kernel-mode is read by more people
famiiliar with WDM than this one.

> I want to port a VxD to WDM because of Win2000. I already read in
> various Websites, but without great success. I bought Walter Oneys
> WDM-Book (seems to be great), but still have problems with my
> work.
> 
> 1) Till now i load my vxd dynamically. Is this still possible ?

If your "driver" is really a kernel-mode extension of an application,
you can use service manager calls to dynamically load and unload it in
Win2K. Check out the PNPMON sample in my book. If it's actually a driver
for hardware, you don't want to be loading and unloading it yourself --
it's better to let the PnP Manager do that (in both OS's)

> 2) Can my new WDM still access parallel port (LPT1) ?

Yes, but not "legally" in the sense that you're not supposed to muck
with ports that belong to another driver.

> 3) Are there any changes with interrupts ? So can i use my
>    old functions inside new WDM ?

Can you be more specific about your needs here? Interrupts work very
differently at the software level in all versions of NT than they do in
Windows, and hardware access also works differently because of the
Hardware Abstraction Layer. Chances are, none of your code will port
very well without major rewrite.

> 4) Are there any new great features for kernel programming ?
>    New functions worth mentioning ?

Hundreds of them, actually. Vielleicht sollen Sie am nächsten April in
meinem WDM Seminar teilnehmen...
