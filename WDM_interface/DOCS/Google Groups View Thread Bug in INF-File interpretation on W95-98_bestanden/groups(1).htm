Hello,

my INF-File has the following section:

[DeviceInstall.nt.LogConfigOverride]
LogConfig=DeviceW2K.LogConfigW2K

[DeviceW2K.LogConfigW2K]
ConfigPriority=NORMAL
MemConfig=2000@0-FFFFFFFF%FFFFF000(RW)
PcCardConfig=1:(A)

On W2K this section should be interpreted, but on W9x
it should be ignored. But on my W98 machine the device
will not be found if the this section is part of my INF-File.
If the section is not in the INF-File evrything is working
all right. The DDKW2K said, if I used the extension "nt"
only W2K will use this section.
Any hints ????

Thanks

The INF parser in Win9x has a fixed limit 31 character for the length of 
section names (look in SETUPX.H in the Win9x DDK). I suspect that it may be 
barfing on your INF file becuase the section name is too long...

The Win2K INF parser does not suffer from this limitation.
